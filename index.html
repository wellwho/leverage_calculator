Fees and funding are ignored. Liquidation price uses MEXC's isolated-margin formula: Avg Entry × (1 + MMR) − Total Margin Deployed ÷ Quantity. Buy sizes grow geometrically (×1.26 per step) so the position scales up as price falls; the ladder is spaced evenly in drawdown across the buys, leaving one spacing unit of buffer before the liquidation target. Buy #1 executes as a market order (fills immediately at the live price); every other buy is a limit order resting at its ladder price.

Orders since your last Execute — not the pair's full trading history. "Filled" rows show how deep this run's ladder actually went; "Resting" ones are still on the book. "Avg entry if filled" is the running position average as if every order down to that row has triggered — a cumulative preview of where your average lands at each rung, same idea as the Ladder table's Avg entry column but built from these real orders. Canceled/invalid orders don't contribute and show "—". While a position is open, "Execute plan on MEXC" is hidden below so you don't accidentally deploy a second ladder on top of it — the Calculate button above still works for previewing numbers only.

Places every "Limit Buy" row above as an isolated-margin, open-long limit order on MEXC Futures (requires MEXC_API_KEY / MEXC_API_SECRET set as environment variables in Vercel). As a last step, whatever of "Available balance" wasn't actually committed to a placed order gets added directly to the position as margin — computed from each order's real, exchange-rounded fill size rather than the plan's theoretical numbers, so it accounts for MEXC's contract-size rounding automatically.
